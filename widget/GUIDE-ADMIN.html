1 Lancer le tableau de bord

Le dashboard est une page HTML locale — elle nécessite un serveur Python pour fonctionner.

1
Ouvrir un terminal Windows
Appuyer sur Windows + R, taper cmd, puis Entrée.
Ou clic droit sur le Bureau → Ouvrir dans le terminal.
2
Se placer dans le bon dossier
Taper la commande suivante puis Entrée :
cd c:\Users\highc\HCS\campaign
3
Lancer le serveur Python
python -m http.server 3000
Le terminal affiche : Serving HTTP on 0.0.0.0 port 3000
⚠️ Ne pas fermer ce terminal — le laisser ouvert en arrière-plan.
4
Ouvrir le tableau de bord dans Chrome ou Firefox
http://localhost:3000/widget/admin-dashboard.html
💡
Pour arrêter le serveur : cliquer dans le terminal et appuyer sur Ctrl + C.

5 Gérer les marchands existants

Consulter, activer, désactiver ou copier le code d'intégration d'un marchand.

📋 Accéder à la liste

Cliquer sur 🏪 Marchands dans la barre latérale. La liste complète s'affiche avec un champ de recherche.

BoutonAction
mrc_xxx (tag bleu) Cliquer pour copier le Merchant ID dans le presse-papiers
👁 Détail Ouvre le détail complet + code d'intégration + options
💻 Code Ouvre directement le générateur de code pour ce marchand
⏸ Désactiver Bloque les paiements du marchand immédiatement (sans supprimer)
▶ Activer Réactive un marchand désactivé
💡
La désactivation est instantanée — le widget du marchand affichera une erreur "Marchand introuvable" dès la prochaine tentative de paiement. Aucun paiement en cours n'est interrompu.

6 Générer le code d'intégration

Produire le code HTML prêt à coller sur le site du marchand.

1
Cliquer sur "💻 Générateur de code" dans la barre latérale
Ou depuis la liste marchands → bouton 💻 Code → le Merchant ID est pré-rempli.
2
Remplir les options
Merchant ID : coller le code mrc_xxx du marchand
Mode : TEST pour les essais, PRODUCTION pour les vrais paiements
Type : Montant fixe / Multi-tarifs / Montant libre
Couleur : couleur principale du bouton (code hexadécimal)
Montant : prix en XPF (entier, sans virgule)
Nom produit / Boutique / Bouton : textes affichés au client
3
Copier le code généré
Cliquer sur 📋 Copier le code. Le code est prêt à coller dans le fichier HTML du marchand.

7 Ce que vous transmettez au marchand

Après création, voici ce que le marchand doit recevoir et comment il l'utilise.

📦 Ce que vous envoyez

ÉlémentExempleUtilité
Merchant ID mrc_aB3xK9pQr Va dans data-merchant-id du HTML
Code HTML prêt Voir ci-dessous À coller dans la page du site du marchand
Mode à utiliser TEST ou PRODUCTION TEST pour valider, puis PRODUCTION pour encaisser

💻 Code à transmettre au marchand

<!-- Widget de paiement — À coller dans votre page HTML -->
<div id="hcs-pay"
  data-merchant-id="mrc_VOTRE_ID"
  data-amount="2500"
  data-product="Nom du produit"
  data-mode="TEST"
  data-color="#6c63ff"
  data-shop="Nom de ma boutique"></div>

<script src="https://payzen-hcs.highcoffeeshirt.workers.dev/widget.js"></script>
✅
Le marchand n'a jamais besoin de ses clés PayZen dans son HTML — seulement le data-merchant-id. Les clés restent chiffrées chez HCS.

8 Dépannage

❌ "Secret incorrect ou Worker inaccessible" à la connexion

!
Vérifier que le serveur Python est bien lancé
Le terminal avec python -m http.server 3000 doit être ouvert et actif.
2
Vérifier le secret
Le secret admin est richesse (tout en minuscules, sans espace).
3
Réinitialiser le secret depuis le terminal
cd c:\Users\highc\HCS\campaign
echo richesse | npx wrangler secret put ADMIN_SECRET

❌ "Migration DB" ne fonctionne pas

Si la migration échoue, exécuter la commande directement depuis le terminal :

cd c:\Users\highc\HCS\campaign
npx wrangler d1 execute hcs-orders-db --file=migrations/002_merchants.sql

❌ "Marchand introuvable" côté client

CauseSolution
Merchant ID incorrect dans le HTMLVérifier la valeur de data-merchant-id
Marchand désactivéDans la liste → bouton ▶ Activer
Migration pas encore faiteCliquer sur ⚙️ Migration DB

❌ Formulaire carte ne s'affiche pas chez le marchand

Le site du marchand doit être en https://. Les iframes bancaires OSB sont bloquées sur les sites en http:// ou en file://.

❌ Paiement TEST accepté mais absent du back-office OSB

En mode TEST, aller dans le back-office OSB → onglet Transactions TEST (pas "Transactions réelles").
