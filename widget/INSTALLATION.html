1 Prérequis

Ce dont vous avez besoin avant de commencer

  • 🏦 Un contrat actif avec OSB Polynésie (Banque de Polynésie) — compte marchand PayZen
  • 🔑 Vos clés d'API REST OSB (onglet "Clés d'API REST" dans le back-office)
  • 🌐 Un site web accessible en https:// (obligatoire pour les paiements)
  • 📧 L'adresse email pour contacter HCS et recevoir votre Merchant ID
⚠️
Le widget ne fonctionne pas si votre site est en http:// non sécurisé ou ouvert depuis un fichier local (file://). Un certificat SSL est obligatoire.

2 Récupérer vos clés OSB Polynésie

Connectez-vous au back-office OSB pour obtenir vos 4 clés d'API

1
Connexion au back-office OSB
Rendez-vous sur le back-office PayZen d'OSB Polynésie. Identifiez-vous avec vos identifiants marchands.
2
Aller dans Paramètres → Clés d'API REST
Dans le menu, cherchez Paramètres → Clés d'API REST (pas les clés V1/V2 — uniquement les clés REST).
3
Copier vos 4 clés
Vous avez besoin de 4 valeurs :
CléExempleUsage
Shop ID 41545085 Identifiant de votre boutique (visible partout dans le back-office)
Clé REST TEST TEST testpassword_xxxx... Pour tester avec des cartes fictives — secrète, ne jamais partager
Clé REST PROD PROD prodpassword_xxxx... Pour les vrais paiements — secrète, ne jamais partager
Clé publique TEST 41545085:testpublickey_... Clé publique test — peut être vue côté client
Clé publique PROD 41545085:publickey_... Clé publique prod — peut être vue côté client
🚫
Ne jamais mettre les clés REST (testpassword / prodpassword) dans votre HTML ou JavaScript. Elles sont transmises uniquement à HCS via un formulaire sécurisé et stockées chiffrées AES-256.

3 Enregistrement chez HCS

HCS crée votre compte marchand et vous remet votre Merchant ID

1
Contacter HCS Tahiti
Envoyez vos informations à HCS : nom de votre boutique, vos 4 clés OSB (via messagerie sécurisée — jamais par email non chiffré).
2
HCS configure votre compte
HCS enregistre vos credentials dans la base sécurisée. Vos clés sont chiffrées en AES-256 avant stockage — même HCS ne peut pas les lire en clair.
3
Vous recevez votre Merchant ID
HCS vous communique un identifiant unique : mrc_abc123xyz
C'est votre seule clé d'intégration. Elle est publique et peut être mise dans votre HTML.
Conservez-la précieusement — c'est elle qui lie votre widget à votre compte OSB.

4 Intégration sur votre site

Collez ces lignes dans votre page HTML — c'est tout

Cas A — Un produit, prix fixe

<!-- 1. Placez ce div là où vous voulez le bouton de paiement -->
<div id="hcs-pay"
  data-merchant-id="mrc_VOTRE_ID_ICI"
  data-amount="2500"
  data-product="Nom de votre produit"
  data-mode="TEST"
  data-color="#6c63ff"
  data-shop="Nom de votre boutique"
  data-label="Commander maintenant"></div>

<!-- 2. Ajoutez ce script juste avant </body> -->
<script src="https://payzen-hcs.highcoffeeshirt.workers.dev/widget.js"></script>

Cas B — Plusieurs tarifs au choix

<div id="hcs-pay"
  data-merchant-id="mrc_VOTRE_ID_ICI"
  data-products='[
    {"name":"Formule Essentielle","price":1500,"description":"Accès 1 mois"},
    {"name":"Formule Pro","price":3500,"description":"Accès 6 mois"},
    {"name":"Formule Annuelle","price":5000,"description":"Accès 12 mois"}
  ]'
  data-mode="TEST"
  data-color="#f093fb"
  data-shop="Nom de votre boutique"></div>

<script src="https://payzen-hcs.highcoffeeshirt.workers.dev/widget.js"></script>

Cas C — Montant libre (acompte, don…)

<div id="hcs-pay"
  data-merchant-id="mrc_VOTRE_ID_ICI"
  data-free-amount="true"
  data-min="500"
  data-max="200000"
  data-product="Acompte sur commande"
  data-mode="TEST"
  data-color="#43e97b"
  data-shop="Nom de votre boutique"></div>

<script src="https://payzen-hcs.highcoffeeshirt.workers.dev/widget.js"></script>
💡
Le script peut être placé une seule fois en bas de page, même si vous avez plusieurs widgets. Tous les <div id="hcs-pay"> de la page seront initialisés automatiquement.

5 Référence des paramètres

Tous les attributs disponibles pour configurer le widget

AttributValeur exempleDescription
data-merchant-id Requis mrc_abc123 Votre identifiant marchand fourni par HCS
data-amount Requis* 2500 Montant en XPF — entier sans décimales
data-product Optionnel "T-Shirt" Nom du produit affiché dans le récapitulatif
data-products Requis* JSON array Liste de tarifs : [{"name":"...","price":1500}]
data-free-amount Optionnel "true" Active la saisie libre du montant par le client
data-min Optionnel 500 Montant minimum autorisé en XPF (défaut : 100)
data-max Optionnel 100000 Montant maximum autorisé en XPF (défaut : 999 999)
data-mode Optionnel TEST TEST (cartes fictives) ou PRODUCTION (vrais paiements) — défaut : TEST
data-color Optionnel #6c63ff Couleur principale du widget en hexadécimal
data-label Optionnel "Payer par carte" Texte affiché sur le bouton principal
data-shop Optionnel "Ma Boutique" Nom de la boutique affiché dans le checkout

* Il faut au moins un parmi data-amount, data-products ou data-free-amount.

6 Tester votre intégration

Validez le fonctionnement complet avant de passer en production

1
Laisser data-mode="TEST"
En mode TEST, aucun vrai paiement n'est effectué. Utilisez uniquement les cartes de test ci-dessous.
2
Ouvrir votre page en HTTPS
Accédez à votre site en https://. Si vous développez en local : http://localhost est aussi accepté.
3
Cliquer sur le bouton de paiement
Le checkout 4 étapes doit s'ouvrir : Panier → Coordonnées → Livraison → Paiement.
4
Utiliser une carte de test
À l'étape Paiement, entrez une des cartes ci-dessous :

❌ Paiement refusé 4970 1000 0000 0015 12/26 123

✓
Validation réussie si…

✅ La commande apparaît dans le back-office OSB Polynésie (section "Transactions TEST")
✅ Aucune erreur dans la console du navigateur (F12)

7 Passer en production

Uniquement après avoir validé les tests avec succès

!
Changer le mode dans votre HTML
Remplacer data-mode="TEST" par data-mode="PRODUCTION" sur tous vos widgets.
2
Vérifier que votre site est en HTTPS
Un certificat SSL valide est obligatoire. OSB Polynésie refuse les origines non sécurisées en mode PRODUCTION.
3
Effectuer un vrai paiement de test
Faites un paiement réel de 100 XPF avec votre propre carte pour valider le circuit complet. Vérifiez que la transaction apparaît dans votre back-office OSB en mode PRODUCTION.
✓
Votre boutique est en ligne
Les paiements de vos clients arrivent directement sur votre compte bancaire OSB Polynésie.
⚠️
Ne jamais tester en mode PRODUCTION avec les cartes de test — elles sont rejetées. En production, seules les vraies cartes bancaires fonctionnent.

8 Questions fréquentes

Les formulaires de carte ne s'affichent pas

Vérifiez que votre page est en https:// ou http://localhost. Si elle est ouverte depuis un fichier (file://), les iframes bancaires sont bloquées par le navigateur.

Le widget affiche "Marchand introuvable"

Votre data-merchant-id est incorrect ou votre compte a été désactivé. Contactez HCS pour vérifier votre Merchant ID.

Le paiement est accepté mais je ne le vois pas dans OSB

En mode TEST, les transactions apparaissent dans l'onglet "Transactions TEST" du back-office, pas dans les transactions réelles. En PRODUCTION, vérifiez l'onglet "Transactions".

Puis-je mettre plusieurs widgets sur la même page ?

Oui. Utilisez des IDs différents : id="hcs-pay-1", id="hcs-pay-2"… ou utilisez la classe class="hcs-payment-widget". Le script est chargé une seule fois.

Mes clés PayZen sont-elles en sécurité ?

Oui. Vos clés secrètes ne transitent jamais dans votre HTML. Elles sont transmises une seule fois à HCS lors de l'enregistrement, puis stockées chiffrées en AES-256 dans une base de données sécurisée. Seul le serveur Cloudflare y accède, jamais le navigateur de vos clients.

Comment modifier le montant ou les tarifs après installation ?

Modifiez simplement les attributs data-amount ou data-products dans votre HTML. Aucune intervention de HCS n'est nécessaire.

Que faire si je veux changer mes clés PayZen ?

Contactez HCS. Vos nouvelles clés seront mises à jour dans la base sécurisée. Votre data-merchant-id ne change pas.
