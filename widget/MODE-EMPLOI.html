💳 HCS Payment Widget

Mode d'emploi complet

Intégrez le paiement OSB Polynésie sur n'importe quelle page web en 2 lignes de code.

https://payzen-hcs.highcoffeeshirt.workers.dev/widget.js
Installation Montant fixe Multi-tarifs Montant libre Paramètres Événements JS Test Production Déploiement

1 Installation

Aucune dépendance, aucun npm. Deux lignes suffisent sur n'importe quelle page HTML.

1
Ajouter un conteneur
Un simple <div> avec des attributs data-* pour configurer le widget.
2
Charger le script
Le widget s'initialise automatiquement au chargement de la page.
3
C'est tout
Le bouton apparaît. Le checkout 4 étapes est prêt. Les paiements arrivent sur ton compte OSB.
⚠️
Le widget requiert une URL https:// ou http://localhost. Il ne fonctionne pas en ouvrant le fichier directement depuis l'explorateur (file://).

2 Cas 1 — Montant fixe

Un produit, un prix. Le client clique, passe commande, paye.

<!-- Conteneur widget -->
<div id="hcs-pay"
  data-amount="2500"
  data-product="T-Shirt Classic Mars"
  data-mode="PRODUCTION"
  data-color="#4facfe"
  data-shop="HCS Tahiti"
  data-label="Commander maintenant"></div>

<!-- Script widget (à placer avant </body>) -->
<script src="https://payzen-hcs.highcoffeeshirt.workers.dev/widget.js"></script>

3 Cas 2 — Sélecteur multi-tarifs

Le client choisit son pack ou sa formule avant de payer. Utilise data-products avec un tableau JSON.

<div id="hcs-pay"
  data-products='[
    {"name":"Sticker A5","price":800,"description":"Impression DTF haute qualité"},
    {"name":"Pack 5 stickers","price":3500,"description":"Économisez 500 XPF"},
    {"name":"Pack 10 stickers","price":6000,"description":"Meilleur prix du pack"}
  ]'
  data-mode="PRODUCTION"
  data-color="#f093fb"
  data-shop="HCS Tahiti"
  data-label="Choisir mon pack"></div>

<script src="https://payzen-hcs.highcoffeeshirt.workers.dev/widget.js"></script>
💡
Chaque objet du tableau peut contenir name (obligatoire), price (obligatoire) et description (optionnel).

4 Cas 3 — Montant libre

Le client saisit lui-même le montant. Idéal pour les acomptes, dons ou devis personnalisés.

<div id="hcs-pay"
  data-free-amount="true"
  data-min="500"
  data-max="200000"
  data-product="Acompte commande personnalisée"
  data-mode="PRODUCTION"
  data-color="#6c63ff"
  data-shop="HCS Tahiti"></div>

<script src="https://payzen-hcs.highcoffeeshirt.workers.dev/widget.js"></script>

5 Tous les paramètres

Attribut Requis Valeur Description
data-amount Requis* 2500 Montant fixe en XPF (entier, sans décimales)
data-product Optionnel "T-Shirt" Nom du produit affiché dans le checkout
data-products Requis* JSON array Multi-tarifs — remplace data-amount
data-free-amount Optionnel "true" Active la saisie libre du montant
data-min Optionnel 500 Montant minimum (free-amount) — défaut : 100
data-max Optionnel 200000 Montant maximum (free-amount) — défaut : 999999
data-mode Optionnel TEST / PRODUCTION Mode paiement — défaut : TEST
data-color Optionnel #6c63ff Couleur principale du widget (hex)
data-label Optionnel "Payer par carte" Texte affiché sur le bouton principal
data-shop Optionnel "HCS Tahiti" Nom de la boutique dans le checkout

* Au moins un de data-amount, data-products ou data-free-amount est requis.

6 Événements JavaScript

Le widget émet des événements pour réagir au résultat du paiement (redirection, confirmation, analytics…).

hcs:success — Paiement accepté

document.getElementById('hcs-pay').addEventListener('hcs:success', function(e) {
  // e.detail = { orderId, amount, product }
  console.log('✅ Paiement accepté', e.detail);

  // Exemple : rediriger vers une page de confirmation
  window.location.href = '/merci.html?order=' + e.detail.orderId;
});

hcs:error — Erreur paiement

document.getElementById('hcs-pay').addEventListener('hcs:error', function(e) {
  // e.detail = { message }
  console.error('❌ Erreur', e.detail.message);
});
ÉvénementDonnées disponiblesDescription
hcs:success orderId · amount · product Paiement accepté par OSB
hcs:error message Erreur technique ou connexion Worker

8 Passer en production

1
Changer le mode
Remplacer data-mode="TEST" par data-mode="PRODUCTION" dans le HTML.
2
Vérifier les clés Cloudflare
S'assurer que PAYZEN_PROD_KEY et PAYZEN_PUBLIC_PROD_KEY sont configurés dans les secrets du Worker.
3
Héberger en HTTPS
La page qui intègre le widget doit être accessible en https://. Les SDKs bancaires refusent les origines non sécurisées.
4
Vérifier dans le back office OSB
Se connecter au back office OSB Polynésie pour confirmer que les transactions apparaissent bien.
⚠️
Ne jamais exposer PAYZEN_TEST_KEY ou PAYZEN_PROD_KEY dans le code HTML ou JavaScript côté client. Ces clés sont uniquement dans le Worker Cloudflare.

9 Mettre à jour le widget

Après avoir modifié hcs-payment-widget.js, déployer en 3 commandes depuis le dossier campaign/ :

$ cp widget/hcs-payment-widget.js public/widget.js Synchronise la source dans public/
$ npx wrangler deploy Déploie le Worker + Assets sur Cloudflare
$ git add -A && git commit -m "update widget" && git push Sauvegarde sur GitHub
🚀
Le widget est disponible immédiatement après le déploiement Cloudflare à l'URL : https://payzen-hcs.highcoffeeshirt.workers.dev/widget.js
