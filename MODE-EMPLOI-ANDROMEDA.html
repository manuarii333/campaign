Chapitre 1
Lancer l'application
A

Option simple — Ouvrir directement

Double-clic sur andromeda-campaign.html dans l'explorateur de fichiers. L'application s'ouvre dans ton navigateur. Toutes les fonctionnalités d'édition et d'export sont disponibles.

Limitation : En mode fichier local (file://), le paiement PayZen ne fonctionne pas à cause du CORS. Utilise le serveur Python pour tester le paiement.
B

Option complète — Serveur Python local (recommandé)

Ouvre un terminal dans le dossier campaign/ et lance :

python -m http.server 3000

Puis ouvre dans ton navigateur :

http://localhost:3000/andromeda-campaign.html
Recommandé pour tester le paiement PayZen, les exports et toutes les fonctionnalités.

Chapitre 3
Sélectionner et éditer une campagne
1

Sélectionner une campagne

Dans la sidebar gauche, clique sur l'une des 8 campagnes. La zone de travail s'actualise avec les paramètres de cette campagne.

Le point vert = campagne marquée "active". Le point gris = inactive. C'est indicatif uniquement, ça n'affecte pas le fonctionnement.
2

Modifications automatiquement sauvegardées

Chaque modification est sauvegardée automatiquement dans le localStorage du navigateur. Fermer l'onglet et rouvrir ne perd pas tes données.

Attention : Vider le cache / localStorage du navigateur efface les modifications. Pense à exporter régulièrement.
Chapitre 4

Titre principal
Le grand texte en haut de l'annonce. Court et accrocheur (max 40 caractères conseillé).
📋
Sous-titre
Description courte sous le titre. Détaille l'offre ou les caractéristiques.

Pastille colorée (ex: "PROMO LIMITÉE"). Attire l'attention. Modifiable en texte + couleur.

3 swatches de couleur prédéfinis par campagne. Clic pour appliquer instantanément.
🖼️
Image de fond
Charger une photo JPG/PNG. Compressée automatiquement (900px, 70% JPEG) avant stockage.
🔲
Opacité overlay
Curseur pour assombrir/éclaircir l'image de fond. Améliore la lisibilité du texte.
!

Changer le format de la vignette

En haut du bloc vignette, clique sur les onglets :

  • 1:1 → Carré — Feed Facebook / Instagram
  • Paysage → 1.91:1 — Feed standard / Audience Network
  • Story → 9:16 vertical — Stories et Reels
!

Logo draggable

Dans la section Logo, charge une image. Elle apparaît en bas à droite de la vignette. Tu peux la glisser-déposer avec la souris pour la repositionner sur l'annonce.

Chapitre 5
Personnaliser la landing page

La landing page est générée automatiquement à partir des données de la campagne. Le bloc d'aperçu en bas du workspace montre la LP en temps réel.

Les éléments synchronisés automatiquement :
  • Titre et sous-titre héro
  • Couleur principale (CTA, prix)
  • Image de fond du héro
  • Logo de la boutique
  • Noms et prix des produits
  • Photos des produits
  • Caractéristiques (3 icônes)
  • Plans / offres (Formation, Abonnements)

Chapitre 6
Exporter les fichiers
1

Exporter la Landing Page HTML

Clic sur 💾 Exporter LP dans la barre de la campagne.

Un fichier andromeda-lp-{id}-{nom}.html est téléchargé. Ce fichier est 100% autonome : images intégrées en base64, CSS inline, widget PayZen inclus. Il peut être envoyé par email ou hébergé n'importe où.

2

Exporter Annonce + LP Fusionné

Clic sur 📤 Exporter tout.

Un fichier andromeda-ad-{id}-{nom}.html est téléchargé. Contient la vignette Facebook + la landing page dans un seul fichier.

3

Exporter toutes les campagnes

Clic sur ⬇️ Exporter tout en haut à droite dans l'en-tête.

Génère un export groupé de toutes les campagnes actives.

4

Utiliser la landing page exportée

  • Ouvre le fichier HTML directement dans un navigateur pour voir le résultat
  • Met-le en ligne sur n'importe quel hébergement (Netlify, GitHub Pages, etc.)
  • Ou publie-le via le Worker (/lp/publish) pour une URL en ligne
  • Intègre l'URL dans ta campagne Facebook Ads comme lien de destination
Conseil : Après l'export, ouvre le fichier dans Chrome et teste le paiement avec la carte de test 4970 1000 0000 0055 (exp: 12/26, CVV: 123).
Chapitre 7
Configurer le paiement

Le paiement est géré par OSB Polynésie / PayZen. Le Worker Cloudflare fait l'intermédiaire entre la LP et l'API OSB.

Dans le panneau d'édition → section "Paiement PayZen" :
ChampDescription
URL WorkerURL complète du Worker : https://payzen-hcs.highcoffeeshirt.workers.dev/payzen-token
Secret WorkerValeur de WORKER_SECRET configurée dans Cloudflare
ModePRODUCTION ou TEST
!

Tester le paiement

  1. Mode = TEST dans le panneau
  2. Exporte la LP et ouvre-la depuis http://localhost:3000
  3. Clique "Commander" → remplis le formulaire client
  4. Paiement : carte 4970 1000 0000 0055 — exp 12/26 — CVV 123
  5. Le paiement doit s'afficher comme Accepté
!

Passer en production

  1. Dans le panneau → changer Mode sur PRODUCTION
  2. Vérifier que PAYZEN_PROD_KEY est configurée dans les secrets Cloudflare
  3. Re-exporter la LP
Important : Ne jamais mettre de vraies coordonnées bancaires avec le mode TEST.

Chapitre 9
Gérer les marchands (multi-tenant)

Le mode multi-marchand permet à d'autres boutiques d'utiliser le widget HCS Pay avec leurs propres identifiants OSB, sans connaître les clés secrètes (cryptées AES-256 côté serveur).

1

Ouvrir le dashboard admin marchands

http://localhost:3000/widget/admin-dashboard.html

Connecte-toi avec l'URL Worker + clé admin.

2

Première utilisation — Migration DB

Si c'est la première fois, va dans Migration → clique Exécuter la migration. Cela crée la table merchants dans D1.

Résultat attendu : "Migration OK"
3

Créer un nouveau marchand (wizard 4 étapes)

  1. Étape 1 — Nom de la boutique
  2. Étape 2 — Shop ID OSB (ex: 12345678)
  3. Étape 3 — Clés REST test + prod (depuis portail OSB)
  4. Étape 4 — Clés publiques test + prod

Un identifiant marchand mrc_xxx est généré automatiquement. Les clés sont chiffrées AES-256 avant stockage.

4

Générateur de snippet

Dans Snippet Generator, sélectionne un marchand → le code HTML d'intégration est généré automatiquement :

<div id="hcs-pay" data-merchant-id="mrc_xxxxxxxxxxxx" data-amount="2500" data-product="Votre produit" data-mode="PRODUCTION"> </div> <script src="https://payzen-hcs.highcoffeeshirt.workers.dev/widget.js"></script>

Copie ce code et envoie-le au marchand. Il n'a qu'à le coller dans son site.

Chapitre 10
Problèmes fréquents
Le paiement ne se lance pas (erreur CORS)

Cause : Tu ouvres le fichier HTML en double-cliquant (file://) au lieu d'utiliser le serveur Python.

Solution : Lance python -m http.server 3000 et accède via http://localhost:3000

Erreur 1101 dans le Worker

Cause : La table D1 n'existe pas (pas encore migrée).

Solution : Dashboard admin → Migration → Exécuter la migration

La clé admin "ne fonctionne pas"

Cause : Le secret ADMIN_SECRET dans Cloudflare est différent de ce que tu saisis.

Solution : Réinitialise le secret :
echo "nouveau_secret" | npx wrangler secret put ADMIN_SECRET

Mes modifications ont disparu

Cause : Le localStorage a été vidé (nettoyage cache navigateur ou autre navigateur).

Solution : Toujours exporter les LP importantes immédiatement après modification. Les fichiers HTML exportés sont permanents.

L'export PNG est flou

Cause : html2canvas est chargé depuis Internet — si hors ligne, ça échoue.

Solution : Assure-toi d'être connecté à Internet lors de l'export PNG.

Le Worker ne répond pas

Diagnostic : Ouvre un terminal et tape :

curl https://payzen-hcs.highcoffeeshirt.workers.dev/

Si erreur : Redéploie avec npx wrangler deploy
